# Contributing

Thank you for your interest in Xynoxa Cloud.

## Quick rules

- Be respectful and constructive
- Keep PRs focused and scoped
- Explain *why* the change is needed
- Add or update docs when behavior changes

## Development

- Do not run builds in this environment unless you know what you are doing
- Prefer Docker Compose for local testing

## Reporting bugs

Use the GitHub issue template and provide:
- steps to reproduce
- expected vs actual result
- logs (if possible)
